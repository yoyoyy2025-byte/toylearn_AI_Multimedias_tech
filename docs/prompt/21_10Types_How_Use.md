## 프롬프트 유형과 선택 이유
프롬프트 유형과 선택 이유
프롬프트 유형: '페르소나 기반 콘텐츠 변환 및 재조정 (Persona-driven Content Transformation & Re-creation)' 프롬프트입니다.

선택 이유:

페르소나 (Persona): '숏폼 콘텐츠 PD'라는 명확한 역할을 부여하여, 단순 요약이 아닌 **'시청자 반응(후킹, 시청 지속 시간)'**을 최우선으로 고려하도록 AI의 목표를 설정합니다.

변환 (Transformation): 원본 데이터(PDF 리포트)를 다른 형식(영상 대본)으로 바꾸는 것이 핵심입니다. '초등학생도 이해하는 수준'이라는 제약 조건은 단순 변환이 아닌, **'의미 보존 및 난이도 하향'**이라는 고도의 변환을 요구합니다.

재창조 (Re-creation): 원본 텍스트에 없는 시각적 요소(화면 구성, 자막, 효과음)를 새롭게 조정 해야 합니다. 이는 AI가 영상 매체의 문법을 이해하고 적용해야 함을 의미합니다.

## 작성 위해 추가 필요 요건
채널 톤앤매너 (Tone & Manner) 구체화: "초등학생 수준" 외에, 채널의 분위기(예: '빠르고 재치 있게', '진지하고 감성적이게')를 명시하면 일관된 결과물이 나옵니다.

핵심 타깃 메시지(TKM) 선정: 60초 안에 모든 쟁점(저작권, 할루시네이션, 편향성)을 다루기 어렵습니다. 가장 강조하고 싶은 **'단 하나의 쟁점'**을 선택하거나, 3가지 쟁점의 우선순위를 정해달라고 지시해야 합니다.

필수 CTA (Call to Action) 지정: 영상 마지막에 시청자에게 유도할 행동(예: '여러분의 생각은 댓글로!', '더보기란의 원본 리포트 확인!')을 명시해야 합니다.

시각적 스타일 가이드: 영상의 시각적 스타일(예: 'AI가 생성한 이미지를 적극 활용', 'PD가 직접 설명하는 토킹 헤드 스타일', '애니메이션 모션 그래픽 위주')을 지정하면 AI가 'visual' 항목을 더 구체적으로 구상할 수 있습니다.

```
출력은 JSON 형식


[참조]
1. 입력 구분 – 원문 분석
2. 변환 목표 설정 – 매체 문법화
3. 출력 포맷 명시 – 스크립트 구조화
4. 평가 기준 제시 – 변환 품질 점검
```
