## 프롬프트 유형과 선택 이유
 - 유형 : 제로샷(Zero-Shot) 역할 기반, 제약 조건 중심의 코드 생성 프롬프트
 선택 이유:제로샷 (Zero-Shot): 이전에 예시를 학습시키지 않고, 단 한 번의 요청으로 복잡한 결과물(웹페이지 전체)을 생성해야 하는 긴급한 상황입니다.
 - 역할 기반 (Role-Based): AI에게 "숙련된 프론트엔드 개발자"라는 역할을 부여하여, 코드의 품질, 구조, 주석 등 전문적인 결과물을 유도합니다.제약 
 - 조건 중심 (Constraint-Driven): "단일 파일", "다크 모드", "Tailwind CSS 사용" 등 명확한 기술적 제약과 "영상 플레이어 영역", "타임스탬프 댓글" 등 구체적인 기능 요구사항을 강력하게 명시해야 합니다. 이는 AI가 상상력을 발휘하는 것을 막고, 정확히 요구되는 프로토타입을 생성하도록 강제합니다.
